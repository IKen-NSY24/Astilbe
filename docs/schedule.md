## 役割分担の原則

| 担当 | 得意領域 | 今回の使い方 |
|------|----------|--------------|
| **あなた（手を動かす）** | 学習価値の高い実装・git・テスト・AWS操作 | ペンツール / AWS構築 / 記事執筆など「身につけたい所」 |
| **Claude Code** | 一括実装・リファクタ・ボイラープレート | 削除作業・定型エンドポイント・詰まった時の肩代わり |
| **Opus 4.8（このチャット）** | 設計判断・コードレビュー・デバッグ相談・記事構成 | 着手前の設計、実装後のレビュー、エラー相談、記事の骨子と推敲 |

### 委譲ルール（迷ったら）
- 見積りの **1.5倍** を超えそう → Claude Code に渡す
- **機械的・反復的**（削除、CRUD雛形、設定ファイル）→ 最初から Claude Code
- **学習目的 or コア体験**（描画ロジック、AWS、記事）→ 自分で。詰まったら Opus に相談

### 最終防衛ライン（全部スリップした時の最小出荷物）
ペン + 長方形 + テキスト + 保存/復元 + デプロイ済み。
（直線・定型文・選択移動は削ってでも「描いて保存して本番で動く」を死守）

# 🟦 Phase 3: インフラ＋アウトプット（〜6/30・火）

### デプロイ構成の決定（重要・先に確定）
> ⚠️ **SQLite はコンテナ系（App Runner / Fargate）だとデプロイ毎に消える**。先に方式を固定する。

- **最小（時間優先・推奨）**: EC2 or Lightsail 1台に NestJS（SQLiteはディスク永続）+ nginx で React静的配信を同居
- **見せる構成（余裕があれば）**: フロント = S3 + CloudFront / バック = EC2（SQLite永続）
